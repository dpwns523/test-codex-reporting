# Codex Reporting Workspace

이 저장소는 Codex에서 바로 사용할 수 있는 보고서 작성용 워크스페이스입니다.
주요 용도는 다음 3가지입니다.

- 금융 보고서
- AML(자금세탁방지), KYC, 제재, 의심거래 관련 보고서
- 추적, 물류 예외, 조사형 타임라인 보고서

이 저장소에는 에이전트 정의와 Codex 설정이 포함되어 있습니다.
참고 자료 파일은 사용자가 이 저장소에 직접 업로드해서 함께 쓰는 방식으로 설계했습니다.

## 포함된 파일

- `AGENTS.md`  
  저장소 전체 동작 규칙
- `.codex/AGENTS.md`  
  Codex 전용 역할 설명
- `.codex/config.toml`  
  Codex 에이전트 등록 설정
- `.codex/agents/report-analyst.toml`  
  범용 보고서 에이전트
- `.codex/agents/finance-reporter.toml`  
  금융 보고서 전용
- `.codex/agents/aml-reporter.toml`  
  AML / KYC / 제재 보고서 전용
- `.codex/agents/tracking-reporter.toml`  
  추적 / 물류 / 조사형 보고서 전용

## 에이전트 역할

### 1. `report_analyst`

가장 범용적인 역할입니다.  
요청이 금융, AML, 추적 중 어디에 정확히 속하는지 애매하거나 여러 주제가 섞여 있을 때 사용하면 됩니다.

### 2. `finance_reporter`

다음과 같은 작업에 적합합니다.

- 재무 요약
- 투자 검토 메모
- KPI 정리
- 리스크 리뷰
- 의사결정용 보고서

### 3. `aml_reporter`

다음과 같은 작업에 적합합니다.

- KYC 검토
- 자금세탁 의심 시나리오 정리
- 제재 스크리닝 결과 정리
- 거래 모니터링 해석
- 컴플라이언스 보고서 작성

### 4. `tracking_reporter`

다음과 같은 작업에 적합합니다.

- 배송 추적 보고서
- 물류 예외 분석
- 캐리어 성과 요약
- 사건 타임라인 재구성
- 조사형 리포트

## 기본 산출물 규칙

모든 요청은 기본적으로 2개의 결과물을 만듭니다.

1. 메인 보고서
2. 분석 보고서

출력 형식을 따로 지정하지 않으면 메인 보고서는 항상 **Word 형식**으로 작성됩니다.

## 지원 형식

### Word

기본 형식입니다. 보통 아래 구조로 작성됩니다.

- 제목
- Executive Summary
- 핵심 사실
- 분석 및 해석
- 권고사항
- 부록 또는 참고자료

### Excel

사용자가 Excel을 요청하면 다음 중 하나로 작성됩니다.

- 시트 구조 설명
- 열 정의
- 표 형식 데이터
- 플래그 또는 계산 로직

현재 환경에서 실제 `.xlsx` 파일을 바로 생성하지 못하는 경우에는, Excel에 곧바로 옮길 수 있는 표와 워크북 설계를 제공합니다.

### PPTX

사용자가 PPTX를 요청하면 다음 중 하나로 작성됩니다.

- 슬라이드별 제목
- 슬라이드별 핵심 메시지
- 근거 데이터
- 권장 시각자료
- 발표 메모

현재 환경에서 실제 `.pptx` 파일을 바로 생성하지 못하는 경우에는, 바로 슬라이드로 옮길 수 있는 데크 구조를 제공합니다.

## 분석 보고서에 반드시 들어가는 내용

분석 보고서에는 항상 아래 내용이 포함됩니다.

- 결과 분석 및 해석
- 왜 이런 구조와 논리로 보고서를 작성했는지에 대한 설명
- 어떤 파일과 어떤 섹션을 근거로 사용했는지에 대한 참조
- 사실 / 추론 / 가정 구분
- 불확실성, 누락 정보, 충돌하는 증거

## 사용 방법

### 1. 저장소 다운로드

```bash
git clone https://github.com/dpwns523/test-codex-reporting.git
cd test-codex-reporting
```

### 2. 참고 자료 업로드

보고서 품질은 입력 자료에 크게 좌우됩니다.
다음 같은 파일을 이 저장소 안에 넣은 뒤 Codex를 실행하면 됩니다.

- 거래 내역 CSV
- 고객 정보 Excel
- KYC 메모
- 조사 대상 문서 PDF
- 배송 추적 로그
- 내부 정책 문서
- 이전 보고서 초안

예를 들어 아래처럼 폴더를 구성해도 됩니다.

```text
input/
cases/
evidence/
references/
output/
```

권장 용도는 다음과 같습니다.

- `input/`  
  원본 데이터와 직접 분석 대상이 되는 파일을 넣는 폴더입니다.  
  예: 거래 CSV, 고객 목록 Excel, 재무제표, 추적 로그

- `cases/`  
  개별 사건이나 조사 단위별 자료를 정리하는 폴더입니다.  
  예: 케이스 메모, 대상별 정리 문서, 사건별 요약본

- `evidence/`  
  근거 자료와 로그를 모아두는 폴더입니다.  
  예: 스크린샷, 원본 로그, 타임라인 메모, 증빙 파일

- `references/`  
  정책, 템플릿, 기준 문서, 보고서 작성 참고자료를 넣는 폴더입니다.  
  예: 내부 정책서, AML 기준 문서, 보고서 양식, 분석 방법론

- `output/`  
  생성된 초안, 최종 보고서, 표, 슬라이드 구조안 등을 저장하는 폴더입니다.  
  예: 보고서 초안, 분석 메모, Excel용 표, PPTX용 슬라이드 아웃라인

처음 시작할 때는 `input/`에 원본 자료를 넣고, 조사 단위가 분리되어야 하면 `cases/`를 추가로 사용하는 방식이 가장 단순합니다.

### 3. Codex 실행

```bash

```

### 4. 에이전트 선택

Codex 안에서 상황에 맞는 에이전트를 사용합니다.

예:

- `/agent report_analyst`
- `/agent finance_reporter`
- `/agent aml_reporter`
- `/agent tracking_reporter`

## 프롬프트 예시

### 금융 보고서 예시

```text
/agent finance_reporter

input/financials-2026-q1.xlsx와 input/management-notes.md를 기반으로
한국어 재무 분석 보고서를 작성해줘.
출력 형식은 Word로 해줘.
메인 보고서와 분석 보고서를 둘 다 만들어줘.
분석 보고서에는 사용한 파일과 섹션명을 정확히 적어줘.
```

### AML 보고서 예시

```text
/agent aml_reporter

input/customer-profile.xlsx, input/transactions.csv, input/kyc-notes.md를 바탕으로
의심거래 검토 보고서를 작성해줘.
출력 형식은 Excel로 해줘.
분석 보고서는 별도의 Word 스타일 메모로 작성해줘.
사실과 추론을 구분해줘.
```

### 추적 보고서 예시

```text
/agent tracking_reporter

input/tracking-log.csv와 input/incident-summary.md를 기반으로
배송 지연 및 예외 상황 타임라인 보고서를 작성해줘.
출력 형식은 PPTX로 해줘.
슬라이드 구조와 발표 메모까지 포함해줘.
분석 보고서에는 어떤 파일의 어떤 섹션을 사용했는지 적어줘.
```

### 범용 보고서 예시

```text
/agent report_analyst

이 저장소 안의 모든 참고 자료를 검토해서
금융 리스크, AML 리스크, 추적 이슈를 함께 다루는 통합 보고서를 작성해줘.
형식을 지정하지 않으면 Word 형식으로 작성하고
별도의 분석 보고서도 반드시 포함해줘.
```

## 좋은 결과를 얻는 방법

- 파일명을 명확하게 정리하세요.
- 보고서 목적을 프롬프트에 분명히 쓰세요.
- 원하는 출력 형식(Word, Excel, PPTX)을 명시하세요.
- 대상 독자(경영진, 감사팀, 조사팀, 고객사 등)를 적어주세요.
- 분석 보고서에서 어떤 수준까지 출처를 남길지 요구사항을 쓰세요.

## 주의사항

- 이 저장소는 보고서 작성용 에이전트 설정 저장소입니다.
- 실제 결과 품질은 업로드한 자료의 정확도와 최신성에 따라 달라집니다.
- AML 또는 제재 관련 문서는 사실 확인과 법률 검토가 필요한 경우가 있으므로, 최종 제출 전에 사람이 검토해야 합니다.
- 형식만 맞춘 보고서보다, 근거 파일이 분명한 보고서가 더 중요합니다.

## 권장 사용 흐름

1. 자료 업로드
2. 적절한 에이전트 선택
3. 출력 형식 지정
4. 메인 보고서 생성
5. 분석 보고서에서 근거와 논리 확인
6. 필요하면 수정 지시 후 재생성
